# DevDocs - Human-Readable Documentation for Contributors

Documentation here is meant for human consumption, as opposed to `aidocs/`, meant for use when 'onboarding' an AI for new PRs.

- `devdocs/new-robot-onboarding-epic.md` - planning slices and guardrails for the v3 onboarding epic
- `devdocs/NewRobot.md` - target transcript for the redesigned single-flow `;new robot` onboarding experience

- `devdocs/proposal1-help-metadata-coding-handoff.md` - proposal notes for command-linked help metadata work
- `devdocs/proposal1-slice0-baseline.md` - baseline notes for the first help metadata proposal slice
- `devdocs/help-fallback-ux-redesign.md` - manager-facing presentation of the v3 help and deterministic fallback UX redesign
- `devdocs/help-sample-transcripts.md` - transcript-driven UX spec for help discovery and mismatched-command recovery
- `devdocs/security_proof.md` - security-control coverage and manual validation notes
- `devdocs/ssh_connector.md` - human-facing SSH connector notes
